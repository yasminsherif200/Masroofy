{# Template : core/history.html ============================ Displays a searchable, tabular list of all transactions in the user's active budget cycle. Edit and Delete actions are conditionally shown only for transactions dated today (matching the server-side eligibility rules enforced by TransactionService). Extends ------- core/base.html Block ----- content A full-width card containing a live-search input and a transaction table, or a "No transactions yet." fallback message. Context Variables ----------------- transactions : QuerySet / list of Transaction All transactions for the active cycle, provided by TransactionDAO.getTransactionsByCycle(). Each object exposes: t.title — short transaction label t.amount — monetary value (float) t.category — category string (may be empty) t.date — date the expense occurred (date object) t.description — optional free-text note t.id — primary key, used in edit/delete URL reversal cycle : BudgetCycle The active cycle instance. Passed for context but not directly rendered in this template. Template Tags / Filters Used ---------------------------- {% load tz %} Loads Django's timezone template tags. {% now "Y-m-d" as today %} Captures the current server-side date as a string (e.g. "2025-05-01") into the variable `today`. {{ t.date|date:"Y-m-d" }} Formats the transaction's date field to the same YYYY-MM-DD string for comparison with `today`. Edit / Delete Visibility Rule ----------------------------- Both action buttons are wrapped in: {% if t.date|date:"Y-m-d" == today %} This means Edit and Delete are only visible for transactions whose date matches today — consistent with the server-side canEditExpense / canRemoveExpense checks in history_view.py. Edit Action ----------- Links to {% url 'edit_transaction' t.id %} (GET request). Delete Action ------------- Submits a POST form to {% url 'delete_transaction' t.id %}. Uses an inline
with {% csrf_token %} so the delete is CSRF-protected and does not rely on JavaScript. Client-side Search (JavaScript) -------------------------------- A plain-text filters visible table rows in real time using an 'input' event listener. For each row the script compares the lowercased full row text against the lowercased search string and toggles row.style.display accordingly. No external libraries required. Notes ----- - The table id="transactionTable" and tbody id="transactionBody" are referenced by the JavaScript filter; do not rename them without updating the script. - Monetary amounts are displayed with the "EGP" suffix hardcoded in the template; currency is not yet dynamic. - The empty-state message ("No transactions yet.") is shown via an {% else %} branch of the {% if transactions %} tag. #} {% extends 'core/base.html' %} {% block content %}
