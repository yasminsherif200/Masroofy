{# Template : core/dashboard.html ============================== Main financial overview for the authenticated user's active budget cycle. Displays key metrics in a responsive card grid and, optionally, a prominent budget-threshold warning banner. Extends ------- core/base.html Block ----- content Heading, optional warning banner, and a 3-column metric grid. Context Variables ----------------- balance : float Remaining unspent budget for the current cycle, rounded to 2 d.p. Provided by BudgetService.getCurrentBalance(). daily_limit : float Safe amount the user can spend per remaining calendar day without exhausting the budget, rounded to 2 d.p. Provided by BudgetService.getSafeDailyLimit(). remaining_days : int Number of calendar days until the cycle end date. Provided by BudgetService.getRemainingDays(). threshold_reached : bool True when ≥ 80 % of the total allowance has been spent. Provided by BudgetService.checkThreshold(). Controls visibility of the orange warning banner. category_data : dict[str, float] (available but not rendered here) Category name → percentage of total spending. Passed in context by home_view.dashboard; currently unused in this template but available for future chart integration. cycle : BudgetCycle (available but not rendered here) The active cycle instance (totalAllowance, start/end dates, etc.). Passed for potential future use (e.g. displaying cycle date range). Layout ------ .grid CSS grid wrapper (defined in style.css) that arranges the metric cards side by side on wider screens and stacks them on mobile. Each .card displays one metric: - Balance (remaining budget in EGP) - Daily Limit (suggested per-day spend cap in EGP) - Remaining Days (days left in the cycle) Notes ----- - All monetary values are rendered with the suffix "EGP"; currency is not yet dynamic (no {{ currency }} variable is injected here, unlike reports.html). - The category_data and cycle context variables are passed but unused; they can be used to add a spending chart directly on the dashboard. #} {% extends 'core/base.html' %} {% block content %}
