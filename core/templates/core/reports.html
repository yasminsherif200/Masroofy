{# Template : core/reports.html ============================ Renders the full spending-analytics report for the user's active budget cycle. Includes: * A 3-card summary grid (total budget, total spent, remaining balance). * A category-breakdown table (category → % of total spending). * A Chart.js chart (data passed via data-* attributes and rendered by the static JS file core/js/charts.js). * A print button (window.print()) for hardcopy reports. Extends ------- core/base.html Block ----- content A full-width card containing all report sections. Context Variables ----------------- cycle : BudgetCycle The active cycle instance. Provided by TransactionDAO.getTotalExpensesByCycle(). balance : float Remaining unspent budget, rounded to 2 d.p. Provided by BudgetService.getCurrentBalance(). category_data : dict[str, float] | None Mapping of category name → percentage of total spending. Provided by BudgetService.calculateCategoryPercentages(). When None or empty, the table and chart sections are replaced by a "No spending data yet." message. transactions : QuerySet / list Full list of transactions for the cycle. Passed in context by reports_view.reports but not currently rendered in this template (reserved for a future detailed transaction table on the reports page). currency : str | None Currency symbol/code to display after monetary values (e.g. "EGP"). NOTE: This variable is referenced in the template ({{ currency }}) but is NOT currently included in the context returned by reports_view.reports. All monetary values will render as " " (with a trailing space) until the view is updated to pass `currency`. Static Assets ------------- chart.js (CDN) Loaded from https://cdn.jsdelivr.net/npm/chart.js when category_data is non-empty. Loaded via {% static 'core/js/charts.js' %}. Chart Data Encoding ------------------- Category labels and percentage values are embedded on the element as pipe-delimited data-* attributes using the |join:',' filter: data-labels="{{ category_data.keys|join:',' }}" data-values="{{ category_data.values|join:',' }}" charts.js should split these strings on ',' to reconstruct the arrays. Template Tags / Filters Used ---------------------------- {% load static %} — enables {% static %} for resolving asset URLs {{ category_data.keys }} — dict_keys view of category names {{ category_data.values }} — dict_values view of percentages |join:',' — joins an iterable into a comma-delimited string Notes ----- - The `transactions` context variable is available but not yet rendered in a table here.
